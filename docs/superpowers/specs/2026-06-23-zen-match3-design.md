# Zen 消消乐 — 课间解压模式设计文档

> 日期：2026-06-23 | 状态：已确认 | 基于现有消消乐项目改造

## 1. 项目概述

### 1.1 核心定位

**纯解压发泄型消除游戏** — 像捏泡泡纸一样，没有失败条件，尽情消除，享受宝石炸裂的爽感。

### 1.2 目标用户

需要在课间 5-10 分钟快速放松的学生/上班族，不想被"会输"的压力困扰。

### 1.3 设计原则

- **零压力**：无限步数，永远不会 Game Over
- **正反馈递增**：连消越多，视觉反馈越强
- **快速进入/退出**：打开就能玩，随时可以关
- **保留挑战**：Zen 为主但不删除原有 Challenge 模式

## 3. Zen 模式游戏逻辑

### 3.1 核心规则

| 特性 | 说明 |
|------|------|
| 步数限制 | **无限制** |
| 时间限制 | **无** |
| 失败条件 | **不存在** — 永远不会 Game Over |
| 无可用交换 | **自动洗牌** — 检测到无可行交换时自动重新生成棋盘 |
| 手动刷新 | "换一局"按钮随时可用 |

### 3.2 追踪指标（替代传统分数）

| 指标 | 显示 | 说明 |
|------|------|------|
| Chain Count | 连消次数 | 当前这波连续消除的计数，每次消除+1，中断归零 |
| Best Match | 最大消除 | 单次消除中消除宝石数量最多的记录 |
| Vibe (满足度) | ✦✦✦ | 情绪化指标：基于连消表现的 1-5 星 |
| Total Cleared | 总消除数 | 本局累计消除了多少颗宝石（可选显示） |

### 3.3 Vibe 满足度算法

```
1-2 连消 → ✦ (calm)
3-4 连消 → ✦✦ (nice)
5-6 连消 → ✦✦✦ (feeling it)
7-9 连消 → ✦✦✦✦ (on fire)
10+ 连消 → ✦✦✦✦✦ (transcendent)
```

### 3.4 自动洗牌逻辑

当 `_hasPossibleMove()` 返回 `false` 时（Zen 模式）：
1. 播放提示文字 "棋盘已清，重新洗牌..."
2. 0.5s 后重新初始化棋盘
3. 不中断任何连消记录

## 4. 特效系统（6 合 1）

### 4.1 粒子爆炸 → 强化现有

- **触发**：每次消除
- **改动**：粒子数量×2、颜色使用深海调色板、添加随机旋转和大小变化
- **动画**：喷射→扩大→缩小→消失，0.5s

### 4.2 屏幕微震 → 新增

- **触发**：单次消除 ≥4 个宝石 或 连消 ≥3 级
- **强度**：2-4px 随机偏移，频率递减
- **动画**：震动 0.2s，幅度线性衰减到 0
- **约束**：只震棋盘容器，不震整个页面

### 4.3 连击大字弹出 → 新增

- **触发**：连消 ≥2 级时（即第二次连续消除开始）
- **内容**："x{N} COMBO!" — N 为当前连消计数
- **动画**：中央弹出(scale 0.5→1.3→1.0) → 停留 0.3s → 向上飘移+渐隐(0.5s)
- **约束**：每次连消只触发一次，新的覆盖旧的

### 4.4 光波扩散 → 新增

- **触发**：每次消除
- **效果**：以消除区域中心为圆心，扩散一个半透明光环
- **动画**：半径 0→棋盘宽度，透明度 0.3→0，持续 0.6s
- **颜色**：取消除宝石的主色调

### 4.5 分数飘字 → 新增

- **触发**：每次消除
- **内容**："+{N}" — N 为该次消除得分（Zen 模式下分数隐藏但飘字保留）
- **动画**：消除位置升起，y 轴位移 -60px，透明度 1→0，持续 0.8s
- **堆叠**：多个飘字不重叠，各自随机 x 偏移 ±20px

### 4.6 边框辉光 → 新增

- **触发**：连消状态驱动
- **效果**：棋盘外层 box-shadow 根据连消级别变色+增强
  - 0 连消：`0 0 20px rgba(59,130,246,0.15)` — 暗蓝
  - 3+ 连消：`0 0 40px rgba(245,158,11,0.3)` — 金色
  - 5+ 连消：`0 0 60px rgba(245,158,11,0.5)` + 呼吸动画
- **动画**：呼吸脉动（亮度周期性变化），不影响棋盘内内容

## 5. 双模式共存设计

### 5.1 模式切换

- 顶部两个 Tab：`🧘 Zen` / `⚔ Challenge`
- 默认启动 Zen 模式
- 切换时无缝重建棋盘，不清除当前模式数据
- 视觉上 Zen Tab 激活时使用深海蓝紫色，Challenge 激活时使用原来红色

### 5.2 模式差异总表

| 特性 | Zen | Challenge (原) |
|------|-----|----------------|
| 步数限制 | ♾️ 无限 | 30 步 |
| 时间限制 | 无 | 可选 60s |
| 失败条件 | 无 | 步数用完 / 时间到 / 无可用交换 |
| 分数 | 隐藏（内部计算飘字用） | 显示 |
| HUD 指标 | Chain / Best / Vibe | 分数 / 步数 / 时间 / Combo |
| 目标分数 | 无 | 5000 分通关 |
| 按钮 | "换一局" | "新游戏" |
